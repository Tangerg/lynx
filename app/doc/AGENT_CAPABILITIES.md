# Agent 能力总览 —— 它能做什么、什么时候用

> **目的**：把 lyra agent 当前具备的能力，按「**是什么 + 什么场景用**」列成一张能力地图，方便判断"这个活该不该交给它、用哪条能力链"。
>
> **方法**：以**代码为准**（后端 `kernel/toolset` 的工具装配 + 协议 RPC 方法面），非照抄记忆。会漂的量（provider 数 / 工具数）不写死，去对应设置面看。基线 **2026-06-23**。
>
> **一句话**：能力分两层——① **Agent 的"手"**：一次运行里 LLM 可调用的工具（读写代码、跑命令、搜索、导航、上网、委派…）；② **围绕运行循环的平台/会话能力**：会话与多标签、中途引导、审批、检查点、定时、配方、钩子、MCP… 第 3 节把它们串成端到端工作流，第 4 节诚实标注边界。

---

## 0. 两层心智模型

```
┌─────────────────────────────────────────────────────────────┐
│  平台 / 会话层（围绕 loop）                                     │
│  会话·多标签·fork │ 中途steer │ 停止 │ HITL恢复 │ 审批模式+规则 │
│  检查点+回滚 │ @codebase索引 │ 定时任务 │ 配方 │ 钩子 │ MCP配置 │
│  多模态 │ 导出导入 │ 记忆 │ 压缩 │ 可观测 │ Provider/模型 │ 持久恢复 │
│   ┌───────────────────────────────────────────────────────┐ │
│   │  Agent 的"手"（一次运行内，LLM 可调用的工具）            │ │
│   │  文件 · shell · 搜索 · 代码智能 · 网络 · 委派 ·          │ │
│   │  技能 · 计划&HITL · 待办 · 外部工具(MCP/A2A)             │ │
│   └───────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

- **手** = agent 在单次运行中"做事"的原语，模型自己决定调哪个。
- **平台层** = 你（或自动化）对运行的控制面 + agent 工作所依赖的基础设施。
- 很多能力是**条件出现**的：缺 key / 没配语言服务器 / 没建索引 / 没装 MCP，对应工具就不挂上（见各条「条件」）。

## 1. Agent 的"手" —— 一次运行中可调用的工具

> 模型在一轮里按需调用。写/执行/联网类工具受**审批模式**门控（见 §2 审批）。标「条件」的，未满足时根本不出现在工具表里。

### 文件系统（按当前会话 cwd 解析）

| 工具 | 是什么 | 什么场景用 |
|---|---|---|
| `read` | 读文本文件的某段（带总行数） | 改之前先读它（写/改有 read-before 守卫）；查看某文件内容 |
| `write` | 新建 / 整体重写 / 追加文件 | 从零建文件、整体替换；盲写已存在文件会被拒（先读） |
| `edit` | 精确替换文件里一段文本 | 改一处 bug / 局部重构；要求 old_string 唯一，否则 `replace_all` |
| `glob` | 按文件名 glob 找路径（`**/*.go`） | "所有测试文件在哪"、按扩展名/目录批量定位 |
| `grep` | 按正则搜文件内容（ripgrep） | "谁调用了 X"、找 TODO/某字符串；支持 content/files/count 三种输出 |

写/改有守卫栈：**路径守卫（禁写 .git 等）→ 每文件串行锁 → read-before → stale 检测 → 改后 LSP 诊断**。

### Shell

| 工具 | 是什么 | 什么场景用 |
|---|---|---|
| `shell` | 跑一条 shell 命令（每次全新 shell，cd/export 不保留） | build / test / git / 装依赖 / 跑脚本；超时自动转后台返回 shell_id |
| `shell_output` | 读后台 shell 的新增输出（可阻塞等待） | 看长 build / dev server / 长任务的进度与结果 |
| `shell_kill` | 停掉一个后台 shell | 结束 dev server、掐掉跑飞的任务 |

### 搜索 / 代码智能

| 工具 | 是什么 | 什么场景用 |
|---|---|---|
| `codebase_search` ｜条件：配了 embedding 模型 | **语义**搜代码（按"意思"不按关键字） | "处理登录的逻辑在哪"——不知道确切符号名时；首用建索引、随文件变更刷新 |
| `lsp` | 语言服务器导航：定义/引用/实现/调用层级/hover/文档符号/工作区符号 | 精准跳转与影响面分析，比 grep 准（理解语义而非文本） |
| `lsp_diagnostics` ｜条件：配了语言服务器 | 取某文件的编译错误/警告 | 改完确认没引入类型错（写/改后也会自动附诊断） |

### 网络（均为条件工具，需对应凭据/白名单）

| 工具 | 是什么 | 什么场景用 |
|---|---|---|
| `web_search` ｜条件：配了搜索 key | 搜网（Tavily/Brave/Exa…） | 查最新用法/报错/文档；**必须按 [标题](URL) 引用来源** |
| `web_fetch` ｜条件：配了 Jina key | 抓取并以 markdown 读单个网页 | 读某篇文档/页面原文（私有/需登录页会失败） |
| `http_request` ｜条件：配了 host 白名单 | 调一次 HTTP（REST/JSON API） | 打第三方 API；受 host + method 白名单约束，响应上限 256KiB |

### 委派 / 技能 / 计划 / 待办

| 工具 | 是什么 | 什么场景用 |
|---|---|---|
| `task` ｜仅 coding 角色（不可递归） | 把自包含子任务丢给**干净上下文**的子 agent | 隔离大搜索 / 并行探索多个方向 / 独立子任务；子 agent 有除 task 外的全部工具，用量计入父预算 |
| `skill` ｜条件：存在 skills | 加载可复用指令包（list/load/load_resource，渐进披露） | 某种专门流程/规范打包复用；项目 `.lyra/skills` 叠加全局 |
| `ask_user` ｜仅 coding 角色 | 向你提问并**暂停**等回答（HITL） | 缺关键信息 / 有歧义 / 要你拍板方向；支持单选多选选项 |
| `exit_plan_mode` ｜条件：配了审批服务 | 计划模式下提交计划、批准后转执行 | 「先想清再动手」：plan 模式只读调研 → 提交计划给你批 → 通过即转 balanced 执行 |
| `todo_write` ｜条件：配了 todo 服务 | 维护本会话的结构化任务清单 | 多步任务让进度可见、不漏步；同一时刻仅一个 in_progress |

### 外部工具（动态挂载）

| 工具 | 是什么 | 什么场景用 |
|---|---|---|
| `<server>_<tool>`（MCP）｜条件：配了 MCP server | 外部 MCP server 暴露的工具 | 接 GitHub / Slack / DB / 自建服务…；可按服务器/按工具开关与白名单，重连热替换 |
| `<agent>`（A2A）｜条件：配了远程 agent | 把活委派给**远程**的另一个 agent | 跨 agent 协作；每个远程 agent 折成一个工具，用量计入父预算 |

## 2. 围绕运行的平台 / 会话能力

| 能力 | 是什么 | 什么场景用 |
|---|---|---|
| **会话 + 多标签 + fork** | 多个会话并存（多 tab）、可从某会话分叉 | 并行推进多个任务；从某点 fork 去试不同方向 |
| **中途引导 steer** | 运行中追加一句话，注入当前轮（`runs.steer`） | 不打断地纠偏/补充——"顺便也改下 X"（composer 运行中即 Steer 按钮） |
| **停止 / 取消** | 中止当前运行 | 跑偏了立刻停（Esc / 停止按钮） |
| **HITL 恢复** | 审批/提问后从**暂停点**继续，不重跑 | `ask_user`/审批/`exit_plan_mode` 回答后无缝接上 |
| **审批模式 + 持久规则** | 全局审批姿态 **Plan / Safe / Balanced / Auto(yolo)** + 细粒度记忆规则 | 控制 agent 不经询问能做多少；"这个命令以后总允许/总拒绝"（按会话/项目/全局 scope）。模式每次 tool gate 实时读取，运行中改也即时生效 |
| **检查点 + 回滚** | 整仓 shadow-git 快照，一键回滚（`sessions.rollback`） | agent 改飞了，回到运行前状态（覆盖 agent 的 shell 改动，不只编辑过的文件） |
| **@codebase 语义索引** | 代码库向量索引（建/查/重建 `codebase.*`） | `codebase_search` 的底座；大仓里按意思找代码 |
| **定时任务 scheduler** | cron 定时触发**无人值守**运行某 prompt | 每天/定期巡检、生成报告、跑例行任务 |
| **配方 recipes** | `/slash` 参数化提示模板（`$ARGUMENTS`/`$1..$9`） | 把常用提示沉淀成命令，一键带参展开 |
| **钩子 hooks** | 生命周期子进程钩子（PreToolUse 拦截/改写/询问、注入上下文、Stop/Notification 观测） | 团队规则强制：某些命令必须拦/改/确认；运行开始注入项目上下文 |
| **MCP 配置** | 注册外部 MCP server + 每工具门控（禁用/自动批准白名单） | 接公司内外部系统能力；细粒度管哪些工具能用、要不要批 |
| **多模态图片输入** | 往对话里贴图（inline base64） | 截图报错、给设计图让 agent 看图办事（需多模态模型） |
| **导出 / 导入会话** | 会话存档与还原（`sessions.export/import`） | 备份、迁移、分享一段会话 |
| **记忆 LYRA.md** | 跨会话长期记忆（`memory.*`） | 记住项目长期约定/偏好，下次自动带上 |
| **压缩 compact** | 上下文接近上限时压缩历史（自动 + `sessions.compact`） | 长会话不爆上下文、保住关键信息 |
| **可观测 OTel** | Traces/Metrics/Logs（生产可换 OTLP exporter） | 排查 agent 行为、看用量/时延、生产监控 |
| **Provider / 模型** | 多家 provider + 模型选择 + utility/embedding 角色 | 按任务挑模型；配嵌入模型（喂 @codebase）、配工具模型 |
| **持久跨重启恢复** | 运行状态类型化快照，重启后可续 | 后端重启不丢进行中的运行/暂停态 |
| **用量与预算** | per-session/汇总用量、子树预算聚合 | 看花了多少 token/钱；子 agent 用量计入父预算并受限 |

## 3. 典型场景 → 能力怎么组合

把上面的原语串成实际工作流——这才是"什么场景用"的答案。

**A. 修一个 bug**
`codebase_search`/`grep` 定位 → `read` 看上下文 → `lsp`(引用/定义) 评估影响面 → `edit` 改 → `shell`(跑 test) → `lsp_diagnostics` 确认无类型错。改飞了用**检查点回滚**兜底。

**B. 实现一个 feature（先计划后动手）**
切 **Plan 模式**（只读）→ 调研（`read`/`grep`/`lsp`/`codebase_search`）→ `exit_plan_mode` 提交计划等你批 → 批准自动转 **Balanced** 执行 → `todo_write` 跟多步进度 → `edit`/`write` + `shell` 验证。

**C. 无人值守的例行任务**
**scheduler** 定时触发 headless 运行某 **recipe** → agent 用 `shell`/`web_search`/`grep` 干活 → 结果落到新会话（侧栏可见）。适合每日巡检/报告。

**D. 接入团队系统**
**MCP 配置** 注册 GitHub/Slack server（按需开关工具）→ agent 用 `github_*`/`slack_*` 建 issue、发通知；高风险工具用**审批模式/钩子**把关。

**E. 并行大型重构**
主 agent 用 `task` 派多个子 agent，各自在**干净上下文**里改一块（互不污染）→ 主 agent 汇总。配 **Safe/Plan 模式** + **检查点** 控制风险。

**F. 查最新文档并落地**
`web_search` 找资料 → `web_fetch` 读原文 → `edit` 落地代码 → `shell` 验证。结果按 [标题](URL) 标来源。

## 4. 边界 / 还没有的（诚实标注）

- **审批模式是全局 stance，没有 per-run 模式**：协议刻意移除了 per-run（`StartRunRequest` 无 mode 字段）；composer 上的选择器是全局 stance 的就近入口，运行中改即时生效，但不是"这一轮单独设"。
- **跨会话「消息内容」全文搜索还没有**：现只有单会话 `items.list` + 语义代码搜索（`codebase_search`）；命令面板的会话标题搜索属待办（ENTRY_POINTS_BACKLOG T3.1a，纯前端可做），消息全文需新增后端搜索 RPC（T3.1b）。
- **媒体输出 / 读图工具（read_media）parked**：现支持图片**输入**，不支持 agent 主动读取/产出媒体文件。
- **无用户鉴权 / 账号 / 多租户**：runtime 是无状态计算单元，协议层零 user 概念；鉴权交给更外层。
- **LLM provider 不做 OAuth**：填 API key，401 让 UI 提示重填。
- `http_request` **默认不可用**：必须显式配 host 白名单才出现（防 SSRF/滥用）。
