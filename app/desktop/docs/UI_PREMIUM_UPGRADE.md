# Lyra UI 高级质感升级分析

> 2024-06-16 · 设计审计文档

---

## 总体评估

Lyra 的设计系统基础**非常扎实**——Linear + Vercel 的融合思路、单一强调色的克制使用、等宽字体作为"技术语言"的惯例、卡片悬浮于画布之上的空间感，这些都远超 95% 的产品。

但要从"优秀设计系统"跃升到"真正有高级感"，差距在于**毫米级的细节决策**。以下按优先级排列所有发现的问题和建议。

---

## 第一优先级：改动小、感知提升极大

### 1. 行高太松，缺乏"工具感"

**现状：** 消息正文行高 1.68（15px 字号时约 25px 一行），body-md 行高 1.55。这是阅读类产品的节奏，不是工程工具的节奏。

**对比：**

| 产品 | 正文行高 |
|------|:---:|
| Linear | 1.45 |
| Vercel 仪表盘 | 1.5 |
| VS Code | ~1.4 |
| **Lyra（当前）** | **1.68** |

**建议：** 消息正文行高从 1.68 降到 **1.5**，body-md（14px）行高从 1.55 降到 **1.45**。整个界面会立刻从"松散"变成"精密"，更像 IDE 而非博客阅读器。

**涉及改动的变量：**
- `--leading-body`：1.55 → 1.45
- 消息内容 `leading-[1.68]` → `leading-[1.5]`
- `body-lg` 行高：1.65 → 1.5

---

### 2. Tab 标签页缺少按压反馈

**现状：** Tab 条有 hover 态（4% 混色背景变化），有 `transition-colors duration-150`，但**没有任何 `:active` 按压态**。

**问题：** 每个 IDE（VS Code、JetBrains）和所有高质量应用，点击 Tab 时都有微小的缩放/颜色反馈，这是最基础的交互完整性信号。

**建议：** 给 Tab 项添加：
- `active:scale-[0.97]` — 微缩放
- `active:transition-transform active:duration-75` — 快速响应

用户每次切换 Tab 都能感受到即时的触觉确认。这是 DESIGN.md §7 已规定但未落实到 Tab 组件的规则。

---

### 3. 表面层级梯度过浅，层次分不开

**现状：** surface-2、surface-3、surface-4 的生成公式是 `color-mix(ink, 5%×N, surface)`。实际效果：

| 层级 | 色值 | 用途 | 问题 |
|------|------|------|------|
| surface-1 | `#181a1d` | 面板 | 基准 |
| surface-2 | `#181a1f` | 悬停/选中行 | 几乎不可见 |
| surface-3 | `#191a21` | 下拉菜单 | 几乎不可见 |
| surface-4 | `#191b23` | 最深浮层 | 几乎不可见 |

四个层级总共只跨越约 2.5 个亮度点。悬停态、选中态应该让人感觉"抬起来了"，而不是"背景稍微变了一点点"。

**建议：** 将 `--depth-step` 从 5% 提到 **8%**。各层级之间就有了可感知的差异，而不会破坏整体克制的调性。

浅色模式下，由于深色文字混入浅色表面感知差异更小，需要将 depth-step 设得更大（**10%**）。

---

### 4. 欢迎页标题过大

**现状：** 欢迎页 h1 是 40px（`text-[40px]`），正文 15px，比例为 2.67×。

**对比：**
- Linear：32px / 15px = 2.13×
- Vercel 产品界面：24px 封顶
- DESIGN.md 定义的 `display-xl`：32px

40px 是营销落地页的尺度，不是产品界面的尺度。

**建议：** 标题从 40px 降到 **32px**，与 DESIGN.md 的 `display-xl` token 对齐。欢迎页会立刻从"宣传页气质"转为"产品界面气质"。

---

### 5. 面板顶部高光线太暗，实际不可见

**现状：** 面板的 `inset 0 1px 0 rgba(255,255,255,0.04)` 顶部内高光——这是 Linear 的标志性手法，让暗色卡片看起来有一层"被上方光源照亮"的顶面。

**问题：** 4% 不透明度在大多数 IPS 屏幕上肉眼不可见，在 OLED/VA 上完全隐形。

**建议：** 提到 6%（`rgba(255, 255, 255, 0.06)`），它会变成一种"隐约可见但不会让人分心"的精致细节。

---

## 第二优先级：做了明显提升，不做也行

### 6. 画布间距不够果断

**现状：** 面板之间的 8px 间距和面板自身的 8px 圆角值相同，导致"圆角的弧度"和"面板间的缝隙"在视觉上融为一体。

**建议：** 要么收紧到 6px（让面板之间更像无缝拼接），要么放开到 10px（让卡片悬浮感更明确）。当前 8px 是一个"两边都不靠"的折中值，缺乏明确的美学立场。推荐 **6px**（匹配 `--radius-sm`），让面板之间的连接感更强——工具类产品更适合紧凑感。

---

### 7. 工具卡片展开动画太慢

**现状：** 工具卡片展开动画使用 `swift` 预设 = 220ms。对于行内内容的展开来说太慢了——用户眼球已经移到下一个元素，动画才刚结束。

**建议：** 缩短到 **140ms**（`dur-fast`）。展开改为干脆利落的快速揭示，而非"悠悠地打开"。220ms 适合模态框/侧边栏，140ms 适合行内内容。

---

### 8. 侧边栏 248px 是个尴尬的中间值

**现状：** 248px 比 Linear 的折叠栏（40px/56px）宽，但比一个舒适导航侧边栏（260-280px）窄。会话标题、项目名、搜索框在这个宽度下都略显拥挤。

**建议：** 扩到 **280px**（匹配 Vercel 侧边栏标准宽度）。在最小 1180px 视口下主区域仍有 900px，远超 760px 的聊天舒适宽度，不存在空间紧张问题。

---

### 9. 墨水层级「淡色」太浅，信息层次模糊

**现状：** muted（`#9ea3ac`）和 faint（`#76787e`）之间的亮度差只有约 14 个点——几乎是同一个层级在重复。实际可感知的墨水层级只有"亮 → 软 → 深"三级，而 faint 应该更后退。

**建议：** 将 ink-faint 从 `#76787e` 压暗到 **`#6b6e74`**。状态栏和元数据的"后退感"才能真正生效，四层墨水梯级各司其职。

---

### 10. 欢迎页缺少"可探索性"

**现状：** 欢迎页只有标题 + 副标题 + 4 个建议按钮，干净但缺乏让用户"探索工具能力"的线索。

**建议：** 在建议按钮下方加一行极小的快捷提示（12px 等宽字体，faint 色），例如：
- ⌘K 打开命令面板
- ⌘B 收起侧边栏
- ⌘N 新建会话

既不妨碍主要功能，又奖励了好奇心，让空白状态更有"引导感"。

---

## 第三优先级：氛围与质感

### 11. 强调色的"活性"表现不足

**现状：** `#1ed760` 是 Lyra 的唯一强调色，DESIGN.md 规定它只能在四种场景出现，其中一种是"动态指示器"（streaming dot、running pill）。但目前动态指示器是纯平的 hex 色，没有发光感。

**建议：** 给运行状态指示灯（状态栏的 ●、Tab 上的小圆点、推理模块的闪烁点）加一层极微弱的 glow——4px 模糊、极低不透明度的 `drop-shadow`。让"运行中"的信号不只是"显示绿色"，而是"微微发光"。这是 Linear 对运行中任务的处理手法——活的暗示。

---

### 12. 画布缺少材质感

**现状：** DESIGN.md 明确禁止了"大气渐变和装饰性网格"，但 `#010102` 这个微蓝调的近黑色是完全平坦的。高端暗色界面（Linear、Arc 浏览器、GitHub Dark）的画布通常有极微弱的纹理——不是渐变，不是图案，而是低于意识感知阈值的"纸张感"。

**建议：** 在画布上叠加一个低于 0.3% 不透明度的噪点纹理（纯 CSS 实现），或者一个极其缓慢的径向微暗角（中心向四周，1-2% 不透明度）。用户不会注意到它的存在，但会感觉到画布"有温度"而非"虚空"。这并不违反 DESIGN.md 的规则——纹理不是大气渐变。

---

### 13. 流式文本到达缺乏质感

**现状：** 流式输出的文字是瞬间出现的。`fade-in` 类用了 0.7 秒 opacity 动画，但对实时流来说太慢，实际上大部分 token 都不会触发这个动画。

**建议：** 给每个 token 一个极微小的到达动画——50ms 内从 `opacity: 0.7 → 1` 并且 `translateY: 1px → 0`。50ms 快到用户感觉不到延迟，但累积起来创造了一种"文字正在被写入"的质感，而非"文字突然出现在那里"。Cursor 的 agent mode 和 Claude Code 都有类似的处理。

---

### 14. 工具卡片选中态太低调

**现状：** `.tool-card.selected` → `bg-line border-line-soft`。和其他工具卡片几乎没区别，但选中的工具卡片驱动着右侧的检测面板（inspector），用户需要一眼看到"哪个工具正在被检视"。

**建议：** 给选中的工具卡片加一条 2px 的强调色左边框（`color-mix(accent, 40%, transparent)`），再叠加一层微弱的强调色内阴影。这是全界面唯一可以适度放宽强调色约束的地方——因为选中的工具就是"活的数据面板"的入口。

---

### 15. 推理模块缺少展开提示

**现状：** 推理模块头部显示 `thinking · 1.2s` 和一个很小的折叠箭头（12px），但头部没有 hover 态变化，不像是可点击的展开控件。

**建议：** 给推理模块头部添加 `hover:bg-surface-3`，让用户感知到"这个区域是可交互的"。

---

### 16. 状态栏数据项缺少 hover 反馈

**现状：** 状态栏是纯展示性的——等宽字体、密集数据、管道分隔符。但像 run ID、分支名、token 用量这些信息，用户可能有复制需求，而当前没有任何 hover 信号表明"这个数据可用"。

**建议：** 状态栏数据项添加 `hover:text-fg cursor-default`，hover 时文字微微变亮。对于可复制的项（run ID、分支名），改为 `cursor-pointer` 并支持点击复制。

---

## 浅色主题专项

### 17. 面板阴影过淡

**现状：** 浅色模式下面板三层阴影的不透明度分别是 6%、7%、8%，在白色画布上几乎看不见。Vercel 的浅色卡片阴影通常在 10-15% 不透明度。

**建议：** 第二层提到 10%，第三层提到 12%。浅色模式下需要更重的阴影才能传达"悬浮"的空间感。

### 18. 表面层级深度步进需要独立调校

**现状：** 深浅模式共用同一个 `--depth-step: 5%`。深色模式下浅色 ink 混入深色 surface 差异明显，但浅色模式下深色 ink 混入浅色 surface 几乎不可见。

**建议：** 浅色模式单独将 depth-step 设为 10%。

## 总结

Lyra 的设计地基是顶尖的——设计理念和克制程度已经超过绝大多数商业产品。阻碍它达到"一眼就感觉高级"的，不是缺失大块功能，而是十几处**毫米级的精细度缺口**：

- **信息密度偏松**：行高太高、标题太大
- **交互反馈不完整**：缺按压态、缺进入/离开过渡
- **表面层级太平**：深度步进太保守、选中态不够明显
- **"活 Agent"的质感不够**：强调色缺少微光、流式文本缺乏到达动画、画布没有材质温度

前 5 个最高优先级的改动，改的都是 CSS 变量和几个类名，不涉及任何功能逻辑，但叠加起来会让整个产品的"手感"从"设计文档写得很好"升级为**"每一个像素都被认真对待"**。
