帮我用 Next.js 写一个带缓存的 fetch 封装，并说明为什么要这么做。
Claude Sonnet 4.5

可以把 fetch 包一层，利用 Next.js 内置的请求缓存。核心思路：

  • 用 next.revalidate 控制缓存生命周期
  • 相同 URL + options 会自动去重（request memoization）
  • 失败时抛出，让上层的 error boundary 兜底
提示：只有服务端组件里的 fetch 才会命中这层缓存。

更多细节可参考 数据获取文档。

export async function cachedFetch(url: string) {
  const res = await fetch(url, {
    next: { revalidate: 3600 },
  });
  if (!res.ok) throw new Error(res.statusText);
